Why this exists. Current spike branch too big to land in one PR. Plan: rewrite from scratch as small, independently mergeable slices behind unused-by-default surface. Each ticket below should be one PR. Order honors dependencies; ticket sizes hint relative effort, not absolute days.

Ground rule. No ticket past NGX-02 needs runtime-mode plumbing, CodeSpaceComponent, or UI. Tickets land dark — feature gated or unreferenced — until NGX-08 wires CodeSpaceComponent in.

Conventions. Each ticket states: Goal, Scope (in/out), Deps, Acceptance, Test plan, Size. Size: S ≤ 1 day, M 2–4 days, L 1 week, XL > 1 week.

NGX-01 NgxScriptSystem skeleton — eval arbitrary JS, console, requestAnimationFrame M no deps

Stand up csp::systems::NgxScriptSystem with a single QuickJS context. Public API exposes only EvaluateSnippet / ExecuteModule on arbitrary text. No asset loading, no components, no UI. Wire host primitives: console.log/warn/error, requestAnimationFrame/cancelAnimationFrame, async/await + Promise microtask pump, periodic GC.

Goal

  • Run untrusted JS strings safely inside foundation.
  • Async/await + native Promises (microtask drain on every tick).
  • console.* routed through LogSystem; Error.stack preserved on uncaught.
  • requestAnimationFrame driven by a display-rate tick separate from foundation tick.

Acceptance

  • EvaluateSnippet("1+1") succeeds; failing snippet logs stack and does not crash.
  • await resolves across a tick boundary (Promise microtask pump runs in PumpPendingJobs).
  • rAF callback fires once per TickAnimationFrame().
  • Context rebuild (OnEnterSpace → OnExitSpace) leaks no JSValues; ASAN clean.

Scope (in)

  • Library/include/CSP/Systems/Script/NgxScriptSystem.h (minimal surface only)
  • Library/src/Multiplayer/NgxScript/NgxScriptSystem.cpp (host bindings only)
  • Tests/src/InternalTests/NgxScriptSystemTests.cpp (subset: bootstrap, rAF, async/await, console capture)

Scope (out)

  • Asset loading, module resolution, code components, UI, materials, entity bindings, input events.

Test plan

  • Port OfflineEnterExitBootstrapsAndTearsDown, TickEventDrainsPendingJobs, RequestAnimationFrameUsesAnimationFrameTickNotFoundationTick.
  • Add: console capture, uncaught rejection logs stack, GC cadence runs every ~180 ticks.
NGX-02 Script assets & module resolution pipeline M deps: NGX-01

Add the module loader. On OnEnterSpace, fetch the space's asset collections, identify script assets, and populate the in-memory LoadedModuleSources map. Install a QuickJS module loader that resolves bare/relative imports against that map. Listen to AssetDetailBlobChanged and refresh affected modules. Add ReloadScriptModule manual path.

Goal

  • Stable canonical module-path mapping (collection + asset name → key).
  • ExecuteModule(path) compiles & runs once; re-imports reuse cache.
  • Asset change events invalidate & recompile.
  • Tracked-asset-collection set prevents reloading unrelated assets.

Acceptance

  • ModuleImportResolvesFromInMemoryMap passes online and offline.
  • ReloadUsesUpdatedSourceAndRerunsEntrypoint passes.
  • MissingModuleLogsExplicitError passes (no silent failures).

Scope (in)

  • NgxScriptSystem: LoadScriptModules, RegisterAssetDetailBlobChangedListener, OnAssetDetailBlobChanged, FetchAssetCollectionMapForSpace, IsTrackedScriptAssetCollection, ReloadScriptModule, InstallModuleLoader.
  • SpaceSystem: call OnEnterSpace/OnExitSpace on NgxScriptSystem.

Test plan

  • Port CanonicalPathMappingIsStable, ExecuteModuleRunsLoadedModuleSource, MissingModuleLogsExplicitError, ReloadUsesUpdatedSourceAndRerunsEntrypoint.

Notes & risks

  • Test ordering. NgxScriptSystemTests.cpp has 70 cases — port them with their owning ticket, not en masse. Acceptance criteria above name the minimum subset per ticket.
  • Hidden coupling. The current branch quietly renames EThirdPartyPlatform values and removes ::Web. That is a breaking enum change; do not bury it inside NGX-04. Split into its own ticket and coordinate with downstream renderer / cspwa.
  • Deletions to defer. The branch removes cmake/, conanfile.py, profiles/host/*, ThirdParty/sole/, and ThirdParty/signalrclient/CMakeLists.txt. None of those are about scripting — leave them out of this roadmap entirely or batch into a "build-system cleanup" PR.
  • Inline JS surface. NgxCodeComponentRuntime.cpp ships a big inline JS string (the scriptRegistry / @csp/code shim). Memory note feedback_csp_code_dual_source tracks the dual-source obligation (C++ inline + ngx-scripts/csp-library). Keep NGX-08 minimal and grow the shim ticket-by-ticket alongside features that need it, rather than landing the full shim up front.
  • Generated-wrapper coupling. Downstream connected-spaces-platform-web-api consumes generated TS. NGX-22 can land first to de-risk every later ticket that adds public API surface.
